# cursor/auto subagent continuation smoke test

## Summary

Added end-to-end smoke coverage and a bridge lookup fix for the provider-qualified `cursor/auto` Cursor MCP tool-result continuation loop:

`mcpArgs â†’ OpenAI tool_calls â†’ role:tool â†’ mcpResult â†’ final-response`

The new tests use `backend.queueLiveRunFrames()` to simulate a Cursor backend that first emits an MCP exec request, then waits until it receives `mcpResult` before sending resumed text and `turnEnded` frames.

Root causes of the observed Sisyphus/cursor-auto confusion:

1. The existing live continuation coverage exercised bare `model: "auto"`, while the real OpenCode cursor provider surface sends provider-qualified `model: "cursor/auto"`.
2. Source already normalizes `cursor/auto` to `auto`, but the publish/runtime entrypoint is `dist/index.js`, so the ignored `dist/` artifact must be regenerated with `bun run build` before a running OpenCode process can use current proxy code.
3. The stronger real-world 409/hang risk was full transcript resume: OpenCode follow-up requests can include historical `role: "tool"` messages from earlier tool calls. `findActiveBridgeKeyByToolCallId()` required every incoming tool result to belong to one paused bridge, so one old tool result plus the current pending `read_file`/subagent result could fail bridge lookup and return `tool_result_continuation_not_found`.
4. The final bridge lookup now accepts a bridge when at least one incoming tool result matches that bridge's pending execs and all unmatched incoming tool IDs are historical (not pending on any other active bridge). It still rejects mixed results from two currently active parallel bridges.
5. Oracle review agreed with the historical-tool matching direction and called out one remaining hang boundary: after a tool result is written, Cursor might stay silent. `handleToolResultResume()` now arms the proxy stream idle timer immediately after attaching the resumed `onData` handler, so a silent post-tool-result stream closes with `[DONE]` instead of hanging indefinitely.
